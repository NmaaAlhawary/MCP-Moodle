What is MCP-Moodle

Two pieces: a Moodle plugin (local_mcpbridge) that adds the activity-creation web services Moodle core is missing, and a Python MCP server that exposes Moodle as 58 typed tools for any AI client. One web service token connects them.

Two pieces, one token

The plugin adds the missing API. The server turns it into AI tools. One token connects them.

  • AI client: Claude Desktop or any MCP-capable client.
  • MCP server: Python bridge with 58 typed tools over Moodle REST.
  • Moodle: your live site, plus the local_mcpbridge plugin.

The mental model

 you talk        MCP server           Moodle
 --------   --tools-->  --------  --REST-->  -------
 "add a quiz"   58 typed tools    real records created

You describe what you want in plain English. The AI picks the right tool, the server calls Moodle's REST API, and real records appear in your live site.

What's in the box

  • local_mcpbridge: a Moodle plugin adding 17 web service functions (quizzes, pages, forums, assignments, sections and more).
  • moodle-mcp: a Python MCP server wrapping core Moodle APIs plus the plugin as 58 tools.
