What it does

Read, build, and stay in control

Moodle's API can create courses and users — but not activities. This project fills that gap and wraps everything as AI tools.

📖

Read

List courses, inspect contents, see enrolled students, quizzes, grades, deadlines, and progress at a glance.

🏗️

Build

Create courses, categories, users, groups, enrolments — plus real content: pages, books, labels, URLs, and quizzes.

🔒

Safely

Every write operation is off by default. A single flag gates all content-changing tools, so nothing happens by accident.

Why it's solid

Built the right way

Upgrade-safe plugin

Uses Moodle's official add_moduleinfo() helper — the same path the web UI uses. Never writes to module tables directly.

Read / write split

Read tools are always on; write tools only register when MOODLE_ALLOW_WRITE=true. No accidental exposure.

Clean errors

Detects Moodle's JSON-error-on-HTTP-200 quirk and surfaces readable messages instead of silent failures.

Security-first

Token never logged, kept out of git, and a full hardening guide for dedicated service accounts on hosted Moodle.

Get started

Up and running in minutes

Install the plugin, generate a token, run the server, connect your AI client.

# 1. Install the plugin on Moodle (Site admin → Install plugins)
# 2. Enable web services + REST, create a token
# 3. Run the MCP server
cd moodle-mcp
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
export MOODLE_URL=https://moodle.example.edu
export MOODLE_TOKEN=your_token
export MOODLE_ALLOW_WRITE=true
python server.py
