Setup

Four steps to get running

Full details in the README.

1

Install the plugin

Site administration → Plugins → Install plugins → upload mcpbridge.zip and confirm the upgrade.

2

Enable web services & create a token

Turn on web services + the REST protocol, then create a token under Manage tokens. This is how the AI authenticates — no password needed.

3

Connect your AI client

Run the Python MCP server with your Moodle URL and token, then add it to claude_desktop_config.json.

4

Start building — just chat

Type what you want: “Create a course called Biology 101 and add a welcome page.” It calls the right tools and does it live.

What it can do

A full course lifecycle

Everything happens directly in your Moodle — real records, not mock-ups. Each card shows an example of what you'd say.

Create courses & sections

Spin up single courses or entire programs, organised into categories and weekly sections.

“Create a Web Dev Bootcamp with 4 courses.”

Add content

Pages, books, labels, URLs, forums, choices and assignments — with rich HTML content.

“Add a Week 1 page and a Q&A forum.”

Build quizzes

Create quizzes and add multiple-choice, true/false and short-answer questions.

“Add a 5-question quiz on cell biology.”

Manage people

Create users, enrol them (single or bulk), unenrol, and build groups.

“Enrol these five students as students.”

Edit & grade

Rewrite page content, move activities between sections, and grade submissions with feedback.

“Grade this submission 85 with feedback.”

Read insights

Grades, deadlines, progress and combined dashboards across all courses.

“What are my upcoming deadlines?”
Compatibility & auth

Works with any Moodle — any login

Runs on any Moodle 4.2+ (self-hosted or institutional). It authenticates with a web service token, not a password — so it doesn't care how your users sign in.

Token-based auth

The AI sends a token an admin generates — never a username or password. It only does what that user's capabilities allow.

Any login method

SSO/SAML, LDAP, Google/Microsoft OAuth, CAS or manual — all work identically. Tokens are a separate auth channel.

What you need

Admin access to install the plugin, enable web services, and create a token. Locked-down managed clouds may not allow custom plugins.
